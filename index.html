Common patterns: full page, hero section with overlay, contain vs cover, repeat, multiple backgrounds.

See the CSS snippets

Full page background (on <body>)

html, body { height: 100%; margin: 0; }
body {
  background-image: url('assets/ironman.jpg');
  background-size: cover;      /* or contain */
  background-position: center; /* top | right | bottom | left */
  background-repeat: no-repeat; /* or repeat */
}

Background on a section with overlay

.hero {
  background-image:
    linear-gradient(to bottom right, rgba(0,0,0,.55), rgba(0,0,0,.35)),
    url('assets/ironman.jpg');
  background-size: cover;
  background-position: center;
}

Contain (show entire image)

.box {
  background: #f1f5f9 url('image.png') no-repeat center / contain;
}

Repeating pattern

.pattern {
  background-image: url('small-tile.png');
  background-repeat: repeat; /* default */
  background-color: #e2e8f0; /* optional fallback */
}

Multiple backgrounds

.box {
  background-image:
    radial-gradient(80px 80px at 20% 30%, rgba(255,255,255,.7), transparent),
    url('ironman.jpg');
  background-size: auto, cover;
  background-position: center, center;
  background-repeat: no-repeat, no-repeat;
}

Quick tips

  • Use background-size: cover for edge-to-edge designs (some cropping may occur).
  • Use contain when you must show the full image (may leave empty space).
  • Add overlays (gradients or semi-transparent layers) to keep text readable.
  • Provide a matching background-color as a graceful fallback.
